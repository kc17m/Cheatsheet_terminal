GitSuperUser erstellt neues Repo als Master. Dann lädt er GitCoWorker zur Zusammenarbeit ein: in gitHub > Repository -> Settings -> Manage Access, (Passworteingabe?), sucht nach GitCoworker. Dieser bekommt einen Link per E-Mail, den er bestätigt. (Ohne Branching: GitCoworker geht in gitHub zum entsprechenden Repository > Code > Clone erstellt im Mac Terminal oder Finder einen neuen Ordner, öffnet diesen in VS Code.)

GitSuperUser und GitCoWorker arbeiten in verschiedenen Branches parallel. GitCoWorker in gitHub: erstellt Clone im Repo über Code > Clone, öffnet Terminal in VS Code: git clone (kopiert Repo-Name) wechselt in Work-Verzeichnis im Terminal: cd Verzeichnisname, ist damit im Terminal im Projekt und kann damit arbeiten. GitCoWorker erstellt neuen Branch über git checkout -b nameseinesbranches. Jetzt kann GitCoWorker den Branch bearbeiten und mit git add . git commit -m "Kommentar"> git push ins Repo pushen. Bei git push bekommt er im Terminal eine Meldung, was genau eingegeben werden muss. Er kopiert diesen Befehl und führt Push aus.

Der Branch ist noch nicht in main integriert GitSuperUser kann die neuen Edits prüfen über compare and pull request GitSuperUser zeigt die Branches im Repo an über New Pull Request
